# NEYO Founder Manual V2 — Plan, Reading Method & Module Order

**Audience:** NEYO founder and school users with no coding experience.  
**Started:** 2026-07-18.  
**Status:** Built progressively, one real module at a time. Dashboard is Module 01.

## Why this manual exists

The NEYO Bible explains the company, architecture, policies, and institutional knowledge. It is not
a button-by-button user manual. This separate Manual V2 will explain every visible module in plain
language:

- who can see it;
- what must be configured first;
- what every card, field, tab and button does;
- what should appear after pressing it;
- what record changes in the database;
- who receives the result next;
- what to do when the expected result does not appear;
- where settings can be changed;
- what a role is not allowed to do;
- mobile, offline, print and safety notes.

## The standard chapter format

Every module chapter will contain:

1. Purpose and real code sources.
2. Roles and visibility.
3. Before-you-start prerequisites.
4. Page tour from top to bottom.
5. Every button/action and expected result.
6. Full example workflow using Kenyan school data.
7. How data connects to other modules.
8. How to change configuration.
9. Error and troubleshooting table.
10. Role-by-role operating checklist.
11. Founder verification checklist.
12. Edit points for future product changes.

## Truth rule

This manual describes the current code. If a button is present but its behavior is confusing or
incorrect, the chapter will identify that honestly as a product issue rather than inventing an
explanation. A chapter is updated when the real UI or workflow changes.
